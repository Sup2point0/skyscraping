## Midgame

Next I want to turn our attention to this $3$-clue lane. It has a very nice structure, with the lane peak in the tail cell and a [low](../glossary.md#low) skyscraper in the [head cell](../glossary.md#head-cell).

<div class="puzzle">

|||||||||
| :-- | :-- | :-- | :-- | :-- | :-- | :-- | :-- |
|     |     |     |     |  3  |     |  5  |     |
|     |     |     |     |     |     | *12*|  3  |

|     |     |  3  |     |     |     |  5  |     |
|**3**|  2  |     |     |     |     |  6  |     |
|     |     |     |     |     |     | *12*|  3  |
|     |     |  2  |     |     |     |     |     |

</div>

With the lane peak in the tail cell, we only have two possibilities to consider:

- $\text{3 | 2 5 \_ \_ \_ \_ 6}$
- $\text{3 | 2 1 5 \_ \_ \_ 6}$

> There can only be 1 more peak after $2$, so that peak must be the $5$-skyscraper.

Let’s consider the former case.

<div class="puzzle">

|||||||||

|     |     |     |     |     |     | *12*|  3  |
|     |     |  2  |     |     |     |     |     |

</div>

This is invalid by [Dangerzone](../cases/dangerzone.md), since now the $2$-clue half-lane will have at least 3 skyscrapers [visible](../glossary.md#visible).

Hence it must be the latter case!
